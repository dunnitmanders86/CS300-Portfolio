# CS300 Portfolio Reflection

## What was the problem you were solving in the projects for this course?

The main problem I was solving was how to efficiently store, organize, and retrieve course information for the ABCU advising program. The program needed to load course data from a file, display courses in alphanumeric order, and allow users to search for a specific course and view its prerequisites. To solve this problem, I analyzed several data structures, including vectors, hash tables, and binary search trees, to determine which one would best support the program's requirements. Throughout the course, I evaluated the runtime and memory usage of each structure and used that information to make design decisions. The final project involved implementing a working solution that could load course data, maintain course relationships, and provide users with an organized way to access course information.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by comparing vectors, hash tables, and binary search trees to determine which data structure would best meet the program requirements. Rather than focusing only on getting the program to work, I learned to evaluate how different data structures affect performance and efficiency. Throughout the course, I analyzed the runtime and memory usage of each structure and considered how well each one supported tasks such as loading data, searching for courses, and displaying courses in sorted order.

Understanding data structures is important because the way data is stored and organized directly impacts how efficiently a program can perform. A solution that works for a small amount of data may not perform well as the amount of data grows. By comparing different approaches, I gained a better understanding of the trade-offs between speed, memory usage, and functionality. This analysis helped me determine that a binary search tree provided the best balance for the advising program because it allowed efficient searches while also maintaining course information in sorted order.


## How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges I encountered was learning how to analyze algorithm efficiency and implement data structures in C++. As someone with limited programming experience, many of the concepts were unfamiliar at first, especially runtime analysis, pointers, and tree structures. There were times when I understood what the program needed to accomplish but struggled to determine the best way to implement the solution.

I overcame these challenges by breaking larger problems into smaller, more manageable steps and focusing on understanding one concept at a time. I regularly reviewed course materials, worked through examples, and practiced applying the concepts in assignments and projects. I also learned a great deal from instructor feedback, which helped me identify areas for improvement and refine my work. As the course progressed, I became more confident in breaking down problems and choosing the right data structure for the job.


## How has your work on this project expanded your approach to designing software and developing programs?

This project reinforced the importance of planning before writing code. Creating pseudocode, evaluating multiple data structures, and analyzing performance helped me understand that software design decisions should be based on program requirements rather than simply choosing the first solution that works. Throughout the course, I realized there is usually more than one way to solve a problem, but some approaches make the program much more efficient and easier to maintain.

Working through these projects also encouraged me to spend more time thinking about program structure before beginning implementation. I now consider how data will be organized, how users will interact with the program, and how different design choices may affect performance as the program grows. I believe this project helped me develop a more thoughtful and structured approach to software design and problem solving.


## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project helped me appreciate the value of writing organized and readable code. I learned the importance of using meaningful variable names, separating functionality into smaller functions, and maintaining a logical program structure. As the projects became more complex, I realized that well-organized code is much easier to understand, troubleshoot, and modify than code that is written without a clear structure.

Working with multiple data structures and building larger programs also showed me the importance of planning for future changes. Writing code in a modular way makes it easier to add new features, update existing functionality, or fix issues without affecting the entire program. These practices make programs easier to maintain, debug, and update as requirements change. As I continue developing my programming skills, I plan to keep focusing on writing code that is maintainable, readable, and flexible enough to grow with a project's needs.
